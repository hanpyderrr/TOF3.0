# 主动调制分离法：透雾单光子 TOF 成像算法思路

> 版本：2026-05-18
> 背景：基于 PF32 32×32 SPAD 阵列，利用可调激光器和电动镜头实现雾/目标分离

---

## 零、PF32 硬件关键约定

**PF32 采用反向 start-stop 模式：**

```
深度换算：ToF = (1023 - bin_index) × 55ps
         distance = ToF × c / 2
```

**含义**：bin 索引越大 → 物理时间越早 → 对应越近的目标。

```
bin 索引:    低  ←───────────────→  高
物理时间:    晚  ←───────────────→  早
距离:       远  ←───────────────→  近

直方图示意：
计数
│
│                         /\   ← 雾峰（近距离，早返回 → 高 bin 端）
│                        /  \
│  ___/\___             /    \
│      目标峰（远，低 bin 端）   bin →
│
0                              1023
```

所有算法中的"早期/晚期 bin"均以此为准。

---

## 一、问题本质

雾天 TCSPC 直方图中有三个叠加成分（注意 PF32 反向 start-stop，雾峰在高 bin 端）：

- **背景噪声**：均匀分布（暗计数 + 环境光）
- **雾散射峰**：光子被雾粒子后向散射，提前返回（高 bin 端），对数正态或 Gamma 分布
- **目标峰**：真实目标反射，高斯分布，位置编码距离（低 bin 端）

现有 `peak_detect.h` 做的是 argmax，雾峰比目标峰高时必然失效。

---

## 二、核心思想：主动调制分离

**利用可调硬件参数作为"激励"，观察直方图"响应"的差异，区分雾和目标。**

关键物理差异：

| 参数变化 | 雾散射响应 | 目标响应 | 差异性 |
|----------|-----------|---------|-------|
| 激光功率（线性区） | 线性增加 | 线性增加 | 无差异 |
| 激光功率（接近饱和区） | 早期峰 pile-up 压缩 | 后期峰影响相对小 | ✅ 有差异 |
| 积分时间 | 线性累积 | 线性累积 | 无差异 |
| **镜头焦距改变** | **几乎不变** | **随焦距强烈变化** | ✅ **最强差异** |

---

## 三、路径 A：焦距扫描差分（空间维度）

### 物理原理

焦距改变时：
- **目标**（点/面反射体，在固定深度 D_t 处）：对焦准时信号最强，失焦则信号下降
- **雾**（分布在整个光路中的体散射体）：全路径上总有雾粒子贡献，对焦距变化不敏感

这与共焦显微镜的深度选择原理相同。

### 算法

```
测量序列（场景静止约 0.5~1s）：
  H₁ = record(focus=F₁, power=P, time=T)    ← 当前焦点
  H₂ = record(focus=F₂, power=P, time=T)    ← 偏移 ΔF

模型：
  H_F(t) = H_fog(t) + α(F) · H_tgt(t)
  H_fog ：雾成分，与焦距无关
  α(F)  ：目标响应因子，取决于 |F - D_tgt|（需标定）

联立解（已知 α₁, α₂）：
  H_fog(t) = [α₂·H₁(t) - α₁·H₂(t)] / (α₂ - α₁)
  H_tgt(t) = [H₁(t) - H₂(t)] / (α₁ - α₂)
  D_tgt    = argmax(H_tgt) × 55ps × 0.299792mm/ps / 2
```

### 优缺点

**优点**：
- 物理原理清晰，不依赖 EM 初始猜测
- 对极端雾天有效（雾峰再高也无法模拟焦距依赖性）
- 分离后可直接做 Gaussian 亚 bin 精度拟合

**限制**：
- 需要标定 α(F) 曲线（P8 电机标定后才可用）
- 场景需静止（~1s）
- α₁ ≈ α₂ 时（ΔF 太小）噪声放大

## 四、路径 B：激光功率调制差分（时间维度，利用 pile-up）

### 物理原理

TCSPC 的核心非线性：**一个激光周期内只能记录一个光子（pile-up 效应）**。

```
高功率时（接近饱和）：
  雾峰在早期 bin（bin ≈ 50），光子率高 → pile-up 严重，峰被压缩
  目标峰在后期 bin（bin ≈ 300），被遮挡概率更高 → 双重压制

低功率时（线性区）：
  H_low ∝ 真实光子率（两成分均无失真）
```

Pile-up 对早期 bin（雾）和晚期 bin（目标）的压制程度不同，这个差异即为分离依据。

### 算法

```
测量：
  H_high = record(power=P_high, time=T)    ← 接近或进入 pile-up 区
  H_low  = record(power=P_low,  time=T)    ← 线性区（P_low ≈ P_high / k）

归一化：
  H_low_scaled(t) = H_low(t) × k           ← 无 pile-up 时应与 H_high 相等

差分信号：
  D(t) = H_low_scaled(t) - H_high(t)
        D(t) > 0 → pile-up 严重区域 → 早期峰 → 雾成分
        D(t) ≈ 0 → 线性区域           → 后期峰 → 目标成分

输出：
  雾峰位置：argmax(D)
  目标峰候选：argmax(H_high) 在 D(t)≈0 的区间内搜索
```

### 优缺点

**优点**：
- 激光功率切换快（< 100ms），实时性好
- 不需要 P8 电机标定
- 早于 P8 即可验证

**限制**：
- pile-up 模型需标定（死时间 τ_dead、重复频率 f_rep）
- 若工作点始终在线性区（功率偏低），差异不显著
- 需要确认 PF32 在当前配置下 pile-up 可控

## 五、路径 C：两者联合（最鲁棒）

```
常规帧（快速，200ms）：
  激光功率调制差分 → 初步估计雾峰位置 t_fog，目标候选区间

低置信度触发（1s）：
  焦距扫描差分 → 精确分离 H_fog, H_tgt

深度估计：
  D = argmax(H_tgt) × 55ps × c/2 + Gaussian 亚 bin 精度修正

置信度：
  conf = H_tgt_peak / (H_fog_peak + H_tgt_peak)
  conf → FeedbackController（驱动激光/电机自适应）
```

## 六、与现有算法对比

| 维度 | argmax（现有） | EM 混合模型 | 主动调制分离（本方案） |
|------|--------------|------------|---------------------|
| 极端雾鲁棒性 | 极差 | 中（局部最优风险） | 强（物理约束） |
| 初始值依赖 | 无 | 强 | 无 |
| 测量帧数 | 1 | 1 | 2（功率法）/ 2+（焦距法） |
| 标定需求 | 无 | 无 | 需要 α(F) 或 pile-up 模型 |
| 延迟 | 极低 | 低 | 功率法 ~200ms，焦距法 ~1s |
| 创新性 | 无 | 已有文献 | 针对 32×32 SPAD 几乎无文献 |

## 九、与 ML 的结合点

主动调制分离法可以和机器学习形成互补：

1. **分离结果作为伪标签**：用物理算法分离出的 H_tgt 作为 ML 训练的地面真值，无需人工标注
2. **ML 补全置信度低的帧**：物理分离置信度低时，ML 接管（混合推理）
3. **ML 学习 α(F) 标定曲线**：用神经网络拟合焦距响应函数，减少标定工作量
4. **物理引导网络**：将雾散射物理模型内嵌到损失函数，减少所需数据量

## 十、与文献的对比（新发现）

文献综述（27 篇）中没有出现**主动调制分离**的思路，已有方法分两类：

| 类别 | 代表文献 | 方式 |
|------|---------|------|
| 单次测量+统计模型 | SPIE 2020（澳大利亚）、Optics Express 2025（火箭军） | EM/DBSCAN 被动分离 |
| 时间门自适应调整 | Adaptive Gating (Paper #12) | 改变时间窗口，硬件支持 |

**主动调制分离是文献空白区**，属于本项目潜在创新点，但也意味着无现成算法可直接借鉴，需要实验验证。

### pile-up 的处理定位

文献把 pile-up 定位为"需要校正的失真"（第一阶段预处理），而本方案将其作为"可利用的雾/目标差分特征"——这是与现有文献的根本区别，也是创新风险所在。

**前提条件**：需要验证 PF32 在当前工作点是否处于 pile-up 显著区间（即计数率是否足够高）。

### 自适应门控（Paper #12）最近似本思想

Thompson sampling 自适应门控通过调整时间窗口获取更多信息——与焦距扫描通过空间选择性获取差分信息属于同一类思想。若 PF32 支持时间门控，该路线值得优先考虑。
