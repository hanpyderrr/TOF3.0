# 登录方式（哪吒 / RK3568）

> 更新：2026-05-19
> 本机（开发机 `ding-NUC12WSHi7`）：`192.168.31.127`（wlo1 WiFi），已在 `dialout` 组，有 `paramiko 2.9.3` / `pyserial 3.5` / `picocom` / `screen`，**无 `sshpass`**。

两台目标机的登录速查。RK3568 串口的踩坑细节见 `docs/rk3568_connection.md`（权威，勿删）。

---

## 一、哪吒 NUC —— SSH（网络通道）

| 项目 | 值 |
|------|-----|
| 地址 | `192.168.31.79`（enp1s0 有线，DHCP 会变；**开发机是 .127/WiFi，勿混**；哪吒 MAC `00:07:32:b6:49:dc`） |
| 账号 / 密码 | `ding` / `1234` |
| 系统 | Linux `5.15.0-1026-intel-iotg` x86_64，hostname `ding-UP-ADLN01` |
| 端口 | 22（默认） |

开发机**无 `sshpass`**，命令行 `ssh` 会卡在交互输入密码。自动化统一用 paramiko：

```python
import paramiko
c = paramiko.SSHClient()
c.set_missing_host_key_policy(paramiko.AutoAddPolicy())
c.connect("192.168.31.79", username="ding", password="1234",
          timeout=8, banner_timeout=8, auth_timeout=8)
i, o, e = c.exec_command("uname -a", timeout=8)
print(o.read().decode())
c.close()
```

人工交互登录（会提示输密码 `1234`）：

```bash
ssh ding@192.168.31.79
```

文件传输：用 paramiko 的 `SFTPClient`（`deploy/` 下脚本即此模式），不要依赖 `scp`+密码。

## 二、RK3568 —— 串口控制台（唯一稳定通道）

| 项目 | 值 |
|------|-----|
| 板型 | ATK-DLRK3568（正点原子），Buildroot，内核 `4.19.232` aarch64 |
| 接口 | USB-UART **CH340**（`1a86:7523`）→ 开发机 `/dev/ttyUSB0` |
| 波特率 / 帧 | **1500000**（1.5M）, 8N1, raw |
| 登录 | 开机自动进 root shell，**无密码**，提示符 `root@ATK-DLRK3568:/#` |
| 5G（已通） | Quectel RM500U-CNV，CDC-NCM usb0；AT 口在**板子侧** ttyUSB0-4 |

### 前置条件：CH340 适配器必须插在开发机上

连接前确认设备存在：

```bash
ls -l /dev/ttyUSB0          # 不存在 = 适配器没插 / 没被识别
lsusb | grep -i 1a86        # 应出现 1a86:7523 (CH340)
```

> 没有 `/dev/ttyUSB0` 时无法连 RK3568：把 CH340 USB-UART 线插到本开发机 USB 口，再重试。

### 连接（标准库 termios，无需 pyserial）

关键坑（务必遵守）：

- **单一持久 fd 收发**，不要用 `cat` 或反复 `open()`；反复开关 CH340 会翻转 DTR/RTS 复位 RK3568。
- `cflag` 必须清 `HUPCL`，并置 `CLOCAL`；否则关闭/打开端口会干扰板子。
- **绝不**在没设 `CLOCAL`/`O_NONBLOCK` 的情况下 `open()` 板子侧 Quectel ttyUSB0-4（会阻塞挂死板子 shell，需物理断电恢复）。
- 串口长命令易丢：单条短命令（<70 字符）+ 唯一 marker 提取，不用 here-doc / 多命令批处理。
- **命令头被吃 2–3 字节**：每条命令前加 **8 空格 padding**，否则 `hostname` 会变成 `tname` 报 `command not found`。自动化批处理可靠版（slow_write + padding + marker + 清终端污染）见 `docs/rk3568_connection.md`。

最小连接示例（完整脚本与说明见 `docs/rk3568_connection.md`）：

```python
import os, time, select, termios
fd = os.open('/dev/ttyUSB0', os.O_RDWR | os.O_NOCTTY | os.O_NONBLOCK)
a = termios.tcgetattr(fd)
a[0] = a[1] = a[3] = 0
a[2] = (a[2] & ~(termios.CSIZE | termios.PARENB | termios.CSTOPB
                 | termios.CRTSCTS | termios.HUPCL)) \
       | termios.CS8 | termios.CLOCAL | termios.CREAD
a[4] = a[5] = termios.B1500000
a[6][termios.VMIN] = 0; a[6][termios.VTIME] = 0
termios.tcsetattr(fd, termios.TCSANOW, a)
time.sleep(0.2); termios.tcflush(fd, termios.TCIOFLUSH)
os.write(fd, b'uname -a\n')
time.sleep(1.0)
print(os.read(fd, 4096).decode('utf-8', 'replace'))
os.close(fd)   # 仅整个会话结束时关闭
```

人工交互登录：

```bash
picocom -b 1500000 /dev/ttyUSB0      # 退出 Ctrl-A Ctrl-Q
```
