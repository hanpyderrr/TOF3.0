The installer on Windows copies the files to a location in Program Files\Photon Force
This folder will generally not have write permission so if you intend using the Python API or a project you'll be wanting to move it to a new location.

But there are some example Python scripts that can be run out of the box to demonstrate the camera working.

To run from the command line change SampleCode to the name of the script you want to run and do:

python3 SampleCode.py



Frames.py 

Reads 2 frames and outputs them. 
By default this will be sent to standard out but once you move the script to a location with write permission you can set write_to_file = True to save to disk.


SW_Histogramming.py
FW_Histogramming.py

These files show how to read a histogram. If the functionality is unlocked in the camera then you can read histograms generated by the firmware (FW_Histogramming.py), otherwise everyone can read histograms generated by the driver (SW_Histogramming.py).

FW_Histogramming.py needs a custom firmware file PF32_USBC_XEM7310_[6087]_Histogramming.bit copied into the local directory.



ExampleCallback.py

This shows how the log messages can be obtained. These messages are also saved to the log file. On Windows this will be in your temp directory
(open Windows Explorer and in the address bar type %temp%). On Linux the log file is saved in the local directory.


SpeedTest.py

This script reads from the camera and calculates the number of frames that were dropped and other analytics such as megabytes and frames per second etc

NumPy is required to run SpeedTest.py

If it is not installed do

pip install numpy





The driver looks for the default firmware files in ..\..\Firmware and the path to the library files is passed in by each script when loading them.

If you want to run a Python script from another location then you'll need either of the following files:

libPF32_API.so  (Linux)
PF32_API.dll (Windows)

The header file PF32_API.h should be in the same directory as the Python script being run.


On Windows it also needs the Qt and Opal Kelly library files:

okFrontPanel.dll
Qt5*.dll
